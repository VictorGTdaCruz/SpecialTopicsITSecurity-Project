	A ideia do código era, incialmente, de trocar o nome de arquivos e pastas importantes para prejudicar a vítima. Posteriormente percebi que o conceito pode ser expandido para um sequestro de informação(ou simplesmente uma ofuscação de informação).
	A renomeação é feita escolhendo uma pasta root no código e usando um for para varrer todas as subpastas e arquivos dentro da path escolhida. Para cada arquivo e subdiretorio, o programa o renomeia para um inteiro entre 0 e 9999999. Como o windows, por padrão, ordena o conteúdo de uma pasta por ordem alfabética, após a renomeação, os arquivos estarão em ordem baseado no tamanho do int escolhido. 
	Um detalhe importante de ressaltar é que todo arquivo possui uma extensão, que ajuda o computador a identificar qual programa usar para abrir o arquivo, .txt ou .doc por exemplo. Após a renomeação, esta extensão nao existirá mais, então para que a vítima abra algum arquivo com sucesso, ela precisará saber que tipo de arquivo era aquele. Devido a renomeação, ela não saberá.
	A segunda parte do código consiste em gravar cada renomeação em um dicionário, tranformá-lo em JSON e enviar para algum local pela rede. Com esse JSON, o autor poderia informar à vítima aonde está localizado e qual o nome do arquivo que ela deseja, já que são armazenados o nome novo de cada arquivo, assim como o antigo.
	Ainda há novas funcionalidades que desejo implementar, mesmo após a data de entrega, como uma melhora no algoritmo de renomeação, como por exemplo, caso um arquivo esteja sozinho em uma pasta, ele será movido para uma pasta acima ou outro lugar que faça sentido, ou até criar extensões para arquivos diferentes das suas originais.
